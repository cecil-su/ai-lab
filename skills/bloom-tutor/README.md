# Bloom Tutor

基于 Bloom 分类法的 AI 一对一教学 skill。用户只需给一个主题名，AI 负责定范围、建计划、教学、记笔记。

## 教学流程

```
/bloom-tutor {主题}
      │
      ├─ 新主题 → 定范围（2-3 轮问答）→ 生成计划 → 学生确认 → 开始教学
      │
      └─ 已有主题 → 复习待巩固 → 从上次位置继续
```

## Bloom 六层级

| 层级 | 认知目标 | AI 角色 | 学生在做什么 |
|------|---------|--------|------------|
| 1 | 记忆 | 讲解者 | 记住关键事实 |
| 2 | 理解 | 倾听者 | 用自己的话解释 |
| 3 | 应用 | 出题者 | 在新场景中解题 |
| 4 | 分析 | 提问者 | 拆解问题找根因 |
| 5 | 评价 | 辩论伙伴 | 判断方案优劣并论证 |
| 6 | 创造 | 评审者 | 从零设计完整方案 |

## 文件管理

每个主题在 `sessions/{topic}/` 下有两个文件，全部由 AI 自动创建和维护：

| 文件 | 作用 |
|------|------|
| `plan.md` | 学习计划 + 进度标记 + 单元教学内容 |
| `notes.md` | 关键领悟 + 待巩固 + 详细学习记录 |

## 使用

```bash
/bloom-tutor http-caching     # 开始新主题 / 恢复学习
/bloom-tutor                   # 查看所有主题进度总览
```

## 学习中可用指令

- **"跳到 [层级]"** — 答对目标层级题目则跳转
- **"太简单了"** — 出高难度题，答对升级
- **"回顾一下"** — 回退巩固
- **"总结进度"** — 掌握情况 + 薄弱点分析
- **"调整计划"** — 查看并修改学习计划

## 文件结构

